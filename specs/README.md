# 仕様書（マスター）

このディレクトリはアプリの**挙動仕様のマスター**です。テストコードと1:1で連動させ、
中長期の開発で過去挙動が意図せず失われていないことを保証します。

## 連動の仕組み

```
仕様項目  SPEC-HOME-001   ←（同じIDで参照）→   E2Eテスト  test('[SPEC-HOME-001] ...')
```

- 各仕様項目に一意な **SPEC-ID** を振る
- E2E/ユニットテストのタイトル先頭に `[SPEC-ID]` を必ず付ける
- `npm run test:trace` が両者を突き合わせ、次を検出する:
  - **仕様にあるがテストが無い**（カバレッジ漏れ）
  - **テストにあるが仕様に無い**（仕様化漏れ）

## SPEC-ID 採番ルール

```
SPEC-<画面>-<連番3桁>
```

- `<画面>`: 画面/機能の識別子（大文字）。例: `HOME` / `LIST` / `FEEDBACK`
- `<連番>`: 画面内で 001 から連番。**一度振ったIDは再利用・欠番の詰め直しをしない**
  （IDが挙動の永続的な参照点になるため）
- 挙動を廃止する場合は項目を削除せず「廃止」と明記し、対応テストも削除する

## ファイル構成

- 画面ごとに `specs/<画面>.spec.md` を1ファイル作る
- 対応するテストは `test/e2e/<画面>.spec.js`（およびユニットは `test/unit/`）

## 仕様項目テンプレート

```
### SPEC-<画面>-<連番>: <一文タイトル>

- **対象**: <画面/要素セレクタ>
- **前提**: <データ・状態の前提。E2Eではフィクスチャ test/fixtures/assets.fixture.js を使用>
- **挙動**: <ユーザー視点で何が起きるか>
- **期待値**: <検証可能な具体値>
- **テスト**: `[SPEC-<画面>-<連番>]`（test/e2e/<画面>.spec.js）
```

## 運用フロー（実装のたびに回す）

1. Claude が実装
2. 実装分の挙動を E2E/ユニットテストに `[SPEC-ID]` 付きで追加
3. 本マスター仕様書に同じ SPEC-ID の項目を追記/改定
4. `npm test`（trace 突合 → unit → e2e）を全緑で確認
