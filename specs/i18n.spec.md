# 多言語（JA/EN）切り替え仕様

- **画面ID**: `I18N`
- **対応実装**: `assets/app.js`（`I18N` 辞書 / `applyI18n` / `initLangSwitch` / `transformData` の言語フィルタ・カテゴリ翻訳）、`pages/feedback.html`（インライン i18n）
- **対応テスト**: `test/e2e/i18n.spec.js`
- **前提フィクスチャ**: `test/fixtures/assets.fixture.js`
  - 日本語アセット5件（id 101–105）＋英語アセット2件（id 201「Meeting Notes Summarizer」/ 202「Pull Request Reviewer」）
  - アセットの言語はタイトルに日本語文字を含むかで判定する

### 言語別の導出値（期待値の根拠）

| 言語 | 表示アセット | 件数 | カテゴリ |
|------|-------------|------|---------|
| JA（既定） | 101–105 | 5 | 資料作成 / 開発支援 / データ分析（3種） |
| EN | 201, 202 | 2 | Document Creation / Development & Code（2種） |

---

### SPEC-I18N-001: ヘッダーのJA/ENトグルで言語が切り替わり永続化される

- **対象**: `#lang-switch [data-l]`、`html[lang]`、`.site-head .nav`
- **前提**: 既定は日本語（localStorage 未設定）
- **挙動**: EN ボタンを押すと言語が英語に切り替わり、`localStorage.aa4h_lang` に保存され、リロード後も維持される。`<html lang>` とナビ文言が英語になる
- **期待値**: クリック後、`localStorage.aa4h_lang` = `en` / `html[lang]` = `en` / ナビ先頭 = `Home`
- **テスト**: `[SPEC-I18N-001]`

### SPEC-I18N-002: 選択言語のアセットのみを表示する

- **対象**: `#grid .card`（`pages/agents.html`）
- **前提**: フィクスチャ（日本語5件＋英語2件）
- **挙動**: 日本語選択時は日本語アセットのみ、英語選択時は英語アセットのみを表示する
- **期待値**: JA=5件、EN=2件。EN時のタイトル集合 = {Meeting Notes Summarizer, Pull Request Reviewer}
- **テスト**: `[SPEC-I18N-002]`

### SPEC-I18N-003: 英語選択時はカテゴリ名を英語表示する

- **対象**: `#chips .chip`（`pages/agents.html`）
- **前提**: 英語選択・フィクスチャ
- **挙動**: `data/` のカテゴリ名（日本語固定）を表示側で英語に差し替える。チップの `data-cat` は元の日本語キーのまま
- **期待値**: `data-cat="資料・文書作成"` のチップ表示に `Document Creation`、`data-cat="開発・コード支援"` に `Development & Code` を含む
- **テスト**: `[SPEC-I18N-003]`

### SPEC-I18N-004: 全ナビ項目を言語に応じて翻訳する

- **対象**: `.site-head .nav > a`（`index.html`）
- **前提**: 英語選択
- **挙動**: 「ホーム／アセット一覧／要望を送る」を含む全ナビ項目を翻訳する
- **期待値**: ナビのリンク文言 = `Home` / `Directory` / `Request`
- **テスト**: `[SPEC-I18N-004]`

### SPEC-I18N-005: 要望フォームを言語に応じて翻訳する

- **対象**: `pages/feedback.html` の静的テキストと JS 生成テキスト
- **前提**: 英語選択
- **挙動**: 見出し・送信ボタン等の静的テキストに加え、Issue タイトルのプレフィックス等の JS 生成テキストも翻訳する
- **期待値**: 送信ボタン = `Submit as Issue`、セクション1見出し = `Type of Feedback`。種類「specific」選択後、`#asset-name` に `X` を入力するとプレビュータイトル = `[Asset Request] X`
- **テスト**: `[SPEC-I18N-005]`
