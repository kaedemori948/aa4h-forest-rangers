# update-dev-env

`updateFormat.md` の修正指示をもとに `dev-environment.html` を部分的に更新するスキル。

## トリガー

- `/update-dev-env`

## 手順

### STEP 1: 修正指示を読む

`.claude/documents/updateFormat.md` を読み、以下を把握する。

- **修正目的**: 何のための修正か、なぜ必要か
- **対象スライド**: どのスライドを修正するか（指定外は一切変更しない）
- **補足**: 追加の禁止事項・制約・背景情報

「対象スライド」の記載が曖昧な場合（例：「P.6」か「ブランチ構成スライド」か）は、次のSTEPでHTMLを確認してから特定する。

### STEP 2: スタイルガイドを読む

`.claude/documents/style.md` を読み、以下を把握する。

- カラーパレット（特に `#E60012` の使用箇所と禁止事項）
- 各コンポーネントのクラス構成
- タイポグラフィの階層ルール
- ファイル末尾の「修正時の禁止事項」セクション

### STEP 3: 現在の HTML を読む

`dev-environment.html` を読む。
STEP 1 で特定した「対象スライド」に対応する `<div class="slide" ...>` ブロックを特定する。

### STEP 4: 修正を実施する

以下の制約をすべて守った上で、対象スライドのみ編集する。

**スコープ制約**
- 対象スライド以外の HTML に一切触れない
- ナビゲーション・カウンター・セクションラベル（`no-print` 要素）は変更しない
- `<script>` ブロックは変更しない
- `<style>` ブロックは変更しない

**スタイル制約**
- `style.md` の禁止事項をすべて守る
- インラインスタイルを追加しない（Tailwind クラスで対応する）
- 新しいカラーコードを追加しない（パレット外の色を使わない）

**内容制約**
- `updateFormat.md` の「補足」に記載された禁止事項・制約を守る
- 指示された変更以外の表現・構造を勝手に改善しない

**判断基準（表現に迷ったとき）**
1. `updateFormat.md` の指示内容を正確に反映する
2. `style.md` のルールに従う
3. 前後の文脈と自然につながる表現にする
4. 余計な語句・要素を追加しない

### STEP 5: 完了報告

修正した内容を以下の形式で報告する。

```
## 修正完了

対象スライド: P.X（スライドタイトル）

| # | 変更箇所 | 変更前 | 変更後 |
|---|---------|--------|--------|
| 1 | ...     | ...    | ...    |

対象外スライド: 変更なし
```

変更できなかった項目がある場合（指示と HTML の不一致など）はその理由も報告する。
