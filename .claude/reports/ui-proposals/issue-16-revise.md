# Issue #16 再改善案：アセット検索・導線改善

## 前回案の問題点

### 却下理由（PRコメントより）
- UIが崩れている
- ボタンが反応しない

### 技術的根本原因
1. **絞り込みロジックのバグ**: `state.cats` はカテゴリタイトル文字列で管理されているが、前回実装の `QF_MAP` に定義した `"営業支援"` はデータに存在しないカテゴリ名。実際のカテゴリは `資料・文書作成`・`データ分析・診断`・`情報収集・調査`・`レビュー・チェック`・`開発・コード支援` など。
2. **カテゴリとタグを同時にセット**: `compute()` の絞り込み条件は `cats AND tags` の AND 結合のため、カテゴリ+タグを同時指定すると結果0件になりやすい。
3. **UI構造の問題**: `.quickfilter-bar` は `.toolbar` の外に配置しているため sticky 位置計算がずれ、レイアウト崩れが発生。

## 新改善案：自然言語チャット型アセット推薦ウィジェット

### コンセプト
既存の `suggestAgents()` 関数（トークンベーススコアリング）を活用し、テキスト入力から「3件以内に絞った推薦カード」をリアルタイム表示する専用 UI パネルをページ上部に追加する。

ユーザーフロー:
1. 「やりたいことを入力」テキストエリアに業務課題を日本語で入力
2. 入力内容をトークン分解し、スコアリングでアセットを上位3件に絞り込む
3. 推薦カードをパネル内に表示（タイトル・カテゴリ・マッチ理由付き）
4. カードクリックで詳細ページへ遷移

### 実装方針

#### HTML変更（pages/agents.html）
- `.toolbar` の直後に `<section class="discovery-panel">` を追加
- テキストエリア + 推薦カード表示エリア（`.discovery-results`）
- クリア / 閉じるボタン
- toolbarと一体化してstickyにしないため、通常フローに配置

#### CSS変更（pages/agents.html の `<style>` 内）
- `.discovery-panel` : サーフェス背景、ボーダー、パディング
- `.discovery-input` : フルワイドのテキストエリア（min-height: 56px）
- `.discovery-results` : flexレイアウトで推薦カードを横並び（モバイルは縦）
- `.discovery-card` : ホバーアニメーション付きカード
- すべてCSS変数でテーマ値を管理

#### JS変更（pages/agents.html の `<script>` 内 — app.js は変更禁止）
- app.js の `renderList()` 完了後に `initDiscovery()` を呼び出す
- `initDiscovery()` 内でテキスト入力 → `DATA.agents` を使ったスコアリング → 上位3件を表示
- app.js の `suggestAgents()` に相当する関数を独自実装（DATA.agents への参照は `window.ASSETS_DATA` 経由で取得済みデータを利用）
- スコアリング: タグ一致(+3)、カテゴリ一致(+2)、タイトル部分一致(+2)、説明一致(+1)の重み付け
- 結果が0件の場合は「条件に合うものが見つかりませんでした」メッセージを表示

### 既存検索との差別化
- 既存: キーワード検索 → 全件から絞り込み → ユーザーが選ぶ
- 新案: 課題を文章入力 → AI的スコアリング → **最大3件に絞って提示**（選択肢を減らす）
