クライアント社内システム UI/UX刷新プロジェクト
このドキュメントを読めば、同じ環境で開発を始められます。

02

プロジェクト概要

目的

クライアント社内システムのUI/UXを刷新する。既存システムの機能はそのままに、操作性・視認性を改善し、業務効率の向上を目指す。

アーキテクチャ

  • フロントエンドのみを実装し、クライアントの既存APIを呼び出して画面を表示する
  • バックエンド・APIはクライアント側が管理（本プロジェクトでは触らない）
  • モックデータと本番API呼び出しを差し替えやすい構造にし、開発中はモックで動作確認する

技術スタック

  • HTML / CSS / バニラJS：フレームワーク不使用。依存関係をゼロに保ち、長期メンテナンス性を優先
  • CSSテーマ値は -- プレフィックスのCSS変数で管理。レイアウトと色は分離して定義する
  • セマンティックタグを使用し、アクセシビリティ・SEOを考慮した構造にする
  • コメントは「なぜそうしているか」のみ記載。何をしているかは書かない
03

開発ツール構成

ツール 役割 備考
Claude Code AIによるフロントエンド実装 CLAUDE.md・Skills・Agentsを設定して使用。GitHubのIssueを読んでコードを自動生成する
VSCode コード確認・Git操作 実装はClaudeCodeに任せ、人間はコードの確認・レビュー・マージに集中する
GitHub ソースコード管理 Issueを起票して要件を管理。ブランチ・PRベースのレビューフローを採用
GitHub Actions 自動テスト・PR制御 テスト・Lint・セキュリティチェックを自動実行。すべてパスしたものだけPR可能にする
04

Claude Code 設定

CLAUDE.md（コーディング規約・プロジェクト情報）

  • クライアントからもらったモックを元に、ClaudeCodeでWEBサイトの叩き台を作成
  • 叩き台をClaudeCodeに読み込ませてCLAUDE.md案を生成 → 手で修正して完成
  • 記載内容：プロジェクト概要・技術スタック・CSSルール・JSルール・コメント規約・自動化パイプラインの説明
  • CLAUDE.mdはリポジトリのルートに配置。ClaudeCodeが毎回参照するため、規約変更時はここを更新する

Skills（ClaudeCodeの追加機能）

  • superpowers：ClaudeCodeの標準機能を拡張するスキルセット
  • frontend-design：UI/UX実装に特化した指示パターンを提供
  • prototype：デフォルト組み込みのプロトタイピング支援スキルを使用

Agents定義ファイルの置き場所

  • 各Agentの定義は .claude/agents/ 配下にMarkdownファイルとして配置
  • Skillsの定義は .claude/commands/ 配下に配置
05

AIエージェント

QA Agent

品質検査

  • UI操作
  • 入力処理
  • JSイベント
  • レスポンシブ
  • エラー処理
出力先 .claude/reports/qa/
Fixer Agent

修正担当

  • 問題一覧を入力として受け取る
  • 原因確認
  • コード修正
  • git diff で説明
入力元 .claude/reports/
Security Agent

セキュリティ監査

  • XSS
  • innerHTML
  • APIキー漏洩
  • localStorage
  • 外部通信
出力先 .claude/reports/security/
06

開発フロー

STEP 01
Issue作成
GitHubに要件をIssueとして起票する
STEP 02
ClaudeCodeが実装
Issueを読んでフロントエンドを実装
STEP 03
自動テスト
GitHub Actionsが自動テストを実行
STEP 04
PR作成
テストパスしたものだけPR可能
STEP 05
確認・マージ
VSCodeでコード確認しマージ

ポイント

  • 実装のトリガーはGitHub Issue。要件をIssueに書けばClaudeCodeが自動で実装を開始する
  • GitHub Actionsでテスト・Lint・セキュリティチェックを自動実行。すべてパスしないとPR不可
  • 人間の作業はIssue起票・コードレビュー・マージのみ。実装・テストはAIと自動化が担う
  • Agentsが並列で品質検査（QA）・修正（Fixer）・セキュリティ監査（Security）を実施
