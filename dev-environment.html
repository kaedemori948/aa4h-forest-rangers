Overview — 01 / 01

プロジェクト
概要

既存システムの機能はそのままに、操作性・視認性を改善し、業務効率の向上を目指す

目的

クライアント社内システムのUI/UXを刷新する。既存システムの機能はそのままに、操作性・視認性を改善し、業務効率の向上を目指す。

アーキテクチャ
  • —フロントエンドのみを実装し、クライアントの既存APIを呼び出して画面を表示する
  • —バックエンド・APIはクライアント側が管理（本プロジェクトでは触らない）
  • —モックデータと本番API呼び出しを差し替えやすい構造にし、開発中はモックで動作確認する
技術スタック
HTML CSS バニラJS CSS変数（--プレフィックス） セマンティックタグ フレームワーク不使用

Claude Code — 01 / 01

Claude Code
設定

CLAUDE.md・Skills・Agentsの3層構造でClaudeCodeの挙動を制御する

CLAUDE.md（コーディング規約・プロジェクト情報）
  • —クライアントからもらったモックを元に、ClaudeCodeでWEBサイトの叩き台を作成
  • —叩き台をClaudeCodeに読み込ませてCLAUDE.md案を生成 → 手で修正して完成
  • —記載内容：プロジェクト概要・技術スタック・CSSルール・JSルール・コメント規約・自動化パイプラインの説明
  • —リポジトリのルートに配置。ClaudeCodeが毎回参照するため、規約変更時はここを更新する
Skills（ClaudeCodeの追加機能）
  • —superpowers：ClaudeCodeの標準機能を拡張するスキルセット
  • —frontend-design：UI/UX実装に特化した指示パターンを提供
Agents定義ファイルの置き場所
  • —各Agentの定義は .claude/agents/ 配下にMarkdownファイルとして配置
  • —Skillsの定義は .claude/commands/ 配下に配置

Agents — 01 / 01

AIエージェント

品質・修正・セキュリティを専門エージェントが並列で担当する

QA Agent
品質検査
  • —UI操作
  • —入力処理
  • —JSイベント
  • —レスポンシブ
  • —エラー処理
出力先
.claude/reports/qa/
Fixer Agent
修正担当
  • —問題一覧を入力として受け取る
  • —原因確認
  • —コード修正
  • —git diff で説明
入力元
.claude/reports/
Security Agent
セキュリティ監査
  • —XSS
  • —innerHTML
  • —APIキー漏洩
  • —localStorage
  • —外部通信
出力先
.claude/reports/security/

Flow — 01 / 01

開発フロー

STEP 01
Issue作成
GitHubに要件をIssueとして起票する
STEP 02
ClaudeCodeが実装
Issueを読んでフロントエンドを実装
STEP 03
自動テスト
GitHub Actionsが自動テストを実行
STEP 04
PR作成
テストパスしたものだけPR可能
STEP 05
確認・マージ
VSCodeでコード確認しマージ

ポイント

→
実装のトリガーはGitHub Issue。要件をIssueに書けばClaudeCodeが自動で実装を開始する
→
GitHub Actionsでテスト・Lint・セキュリティチェックを自動実行。すべてパスしないとPR不可
→
人間の作業はIssue起票・コードレビュー・マージのみ。実装・テストはAIと自動化が担う
